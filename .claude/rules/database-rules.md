# データベース ルール（課題要件より）

## テーブル設計
- テーブルは最低 2 つ、それぞれに主キー（PK）を持たせる
- テーブル間に外部キー（FK）を最低 1 つ設ける
- 多対多リレーションは中間テーブルで解決する（未解決のまま放置しない）
- 適切なデータ型を使う（TEXT, INTEGER, REAL など）
- CHECK 制約を最低 1 つ追加する

## 必須 SQL コマンド
以下をすべて使用すること：
- `SELECT` に `LIMIT` と `OFFSET` を組み合わせる（ページネーション）
- `INSERT`
- `WHERE`
- `JOIN`

## スターターデータ
- テスト用の初期データを DB に入れておく
- `database-setup.sql`（または `db-setup.sql`）にテーブル作成・初期データ挿入クエリをすべて記載する
- このファイルだけで DB をゼロから再構築できるようにする

## セキュリティ
- ユーザー入力を含む全クエリでプリペアドステートメントを使う
- `node:sqlite` の `DatabaseSync` を使い、`express/database/db.js` 経由でアクセスする
