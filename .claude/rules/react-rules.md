# React ルール（課題要件より）

## アーキテクチャ
- **シングルページアプリケーション（SPA）** にする
- React Router や他のページ遷移ライブラリは**使用禁止**
- ページ遷移が必要な場合は自前で実装する
- コンポーネントは意味のある単位に分割する（UI の役割ごとに分ける）

## 状態管理・コンポーネント間通信
- `useState` で状態を管理する
- 親から子へのデータ受け渡しは `props` を使う
- 状態は適切なコンポーネントに置く（必要以上に上に持ち上げない）

## フォーム
- フォームは**コントロールドコンポーネントパターン**で実装する
  ```js
  const [value, setValue] = useState('');
  <input value={value} onChange={e => setValue(e.target.value)} />
  ```
- 最低 1 つのフォームを実装する

## UI・UX
- ローディング中は視覚的なインジケーターを表示する
- エラー時はエラーメッセージを表示する
- 成功時は成功メッセージを表示する
- 生の JSON データをユーザーに見せない（必ず整形して表示する）
- 基本的な CSS スタイリングを適用する

## 禁止事項
- 外部のコンポーネントライブラリを使わない（MUI, Chakra UI, Bootstrap React など）
- コンポーネントはすべて自分で書く
- 授業で扱っていないライブラリは教授の承認なしに使わない

## ビルド
- Parcel でビルドし、出力先は `express/public/` にする
- ビルドコマンド: `cd react && npm run dev`
